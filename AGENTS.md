# AGENTS.md

## Project

X-CSL-Updater is a Qt 5 / CMake desktop application. The main executable target is `X-CSL-Updater`.

The app downloads X-CSL package indexes, compares them with local files, and updates selected files.

## Source Map

- `CMakeLists.txt` - root CMake entry; generates `src/Info.h`.
- `src/CMakeLists.txt` - Qt executable target and install hook.
- `Description.cmake` - app metadata and version.
- `Install*.cmake` - deploy/package scripts.
- `src/main.cpp` - startup and logging.
- `src/mainwindow.*` - main UI and CLI options.
- `src/IndexStep.*` - index download/parse/check logic.
- `src/UpdateStep.*` - file update/delete/download logic.
- `src/AltitudeDefs.*` - URL and local path mapping.

Do not edit `src/Info.h`; it is generated by CMake.

## Local Build Dirs

Treat these as generated/local output, not source:

- `Project_vs2019x64/` - Visual Studio project generated by `project_vs2019x64.bat`.
- `build/` - Visual Studio build dir used by `win-build-release.bat`.
- `cmake-build-debug/` - CLion Debug build dir.
- `cmake-build-release/` - CLion Release build dir.
- `bin/`, `Package/` - executable/package outputs.

Current CLion profiles are `Debug` and `Release`. Both use:

```text
-DCMAKE_PREFIX_PATH="C:/Qt/5.15.2/msvc2019_64"
-DCMAKE_INSTALL_PREFIX=../Package
-DCLEANUP_INSTALL_FIRST=ON
```

The current CLion CMake caches use `Visual Studio 16 2019`, so use `--config ...` when building them. If stale files like `build.ninja` remain in a build dir, trust `CMakeCache.txt` over filename leftovers.

## Commands

Generate the usual Visual Studio 2019 project:

```powershell
.\project_vs2019x64.bat
```

Build the CLion Debug profile:

```powershell
cmake --build cmake-build-debug --config Debug --target X-CSL-Updater
```

Build the CLion Release profile:

```powershell
cmake --build cmake-build-release --config Release --target X-CSL-Updater
```

Build and package Windows Release with the existing script:

```powershell
.\win-build-release.bat
```

Install/package is Release-only:

```powershell
cmake --build cmake-build-release --config Release --target INSTALL
```
